## 继承机制
父类：描述逻辑，一般定义为抽象类（成员函数virtual）
子类：具体
## 泛型机制
关心关系的实现和存储（不关心类型
1. 函数模板（语法：`template <class T>`，用T表示类型）
2. 类模板（`template <class T>`同上，用T表示泛型，在每一个类函数/类模板前都要template一遍。类模板函数成员自动变成函数模板）
## const机制
对函数参数or整个函数的修饰
常量定义
参数前加const保证参数只读
成员函数后加const变只读（常量成员函数，非/常对象能否调用等）
&引用 节省空间/运算
```
void foo(int & x);
//传参不能传foo(3)
```
## 异常处理
开发者通过throw来抛出异常（`if(i<0) throw errType1();`），具体处理方式由用户决定（通过try来截获异常，在自己程序里写相应处理方法）
```
try
{ //不同情况下调用 }

catch(errType1)
{ //(具体处理) }
```
